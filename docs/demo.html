SmartE-Cycle — Demo Guide

Step 0: Preparation before demo

cd ~/Desktop/smartecycle
.\venv\Scripts\Activate.ps1   # Windows PowerShell

pip install -r requirements.txt

Ensure dataset is ready, model is trained (ml/model.h5 exists), and all folders (ml/data, backend, frontend, simulation, docs) are present.

Step 1: Start Backend Server

cd backend
python app.py

Confirm Flask server is running on: http://127.0.0.1:5000

Backend handles inference (/infer) and pickups (/pickups).

Step 2: Open Frontend

cd frontend

Open index.html in a browser (Chrome/Edge recommended).

Or serve via Python HTTP server:

python -m http.server 8000

Open browser: http://127.0.0.1:8000

Step 3: Upload & Classify Image

On frontend page:

  • Click Choose File, select e-waste image (battery, PCB, etc.)
  • Click Classify & Schedule Pickup

Example dummy images (e-waste items):

Battery

PCB

Phone

Cable

Observe prediction JSON:

{
  "class": "battery",
  "confidence": 0.87
}

Explain predicted class and confidence score.

Step 4: Schedule Pickup

After classification, confirm popup to schedule pickup.

Check backend/pickups.json or GET request /pickups: 

[
  {
    "class": "battery",
    "confidence": 0.87,
    "address": "Demo Address"
  }
]

Explains that pickup is scheduled in simulation.

Step 5: Show Routing Simulation

Open simulation/route_comparison.png:

  • Dashed line: Naive route
  • Solid line: Greedy route
  • Red dot: Start point

Explain: Greedy route reduces travel distance → efficient pickups. Can later use OR-Tools for optimal routing.

Step 6: Show ML Evaluation

Open ml/confusion_matrix.png and explain accuracy, misclassifications, and harder classes.

Optional: Run python ml/eval.py to see classification report in terminal.

Explain metrics: precision, recall, F1-score.

Step 7: Show UML & Architecture

Open diagrams from docs/ folder:

  • usecase.png — actors & actions
  • class.png — data model & relationships
  • sequence.png — interaction flow
  • activity.png — swimlane workflow

Explain system flow: user upload → ML prediction → pickup scheduling → routing → recycling.

Step 8: Conclude Demo

Discuss limitations:

  • Small dataset for demo
  • CPU training is slow
  • Routing is greedy, not optimal

Highlight future features:

  • IoT-enabled bins for automatic detection
  • OR-Tools VRP solver for efficient routing
  • Blockchain traceability for e-waste tracking

        ↓

        ↓
Frontend shows result
        ↓

        ↓
